Generics: 
=========

Generics will allow the classes, interfaces and methods to work with different data 
types while providing compile-time type safety. 

Typesafety: Typesafety in generics means errors caused by using the wrong data type
            are detected at compile time. This prevents ClassCastException at
            runtime. 
            
Generics works only with reference types 
for eg: Printer<int>  - wrong primitives not allowed
        Printer<Integer> -correct
        
The type parameter names L:

 ArrayList vs LinkedList
 -----------------------
 ArrayList uses a dynamic array internally
 LinkedList uses a doubly linkedlist internally. 
    - nodes connected with previous and next references
    
 ArrayList stores elements in continuous memory locations
 Linked stores elements as seperate nodes lined together
 
 ArrayList is fast for searching - it provides fast random access by using
    get(index) method
 LinkedList slower for access because it must traverse 
 the nodes sequentially
 
 ArrayList insertion/deletion in the middle is slow because
 elements must shift
 LinkedList insertion/deletion is faster because 
 shifting is not required and only the links are updated.
  
 ArrayList uses less memory
 LinkedList uses more memory because each node 
  stores data + two references
 
ArrayList mainly implements List interface
LinkedList implements List,Queue and Deque

ArrayList supports all list operations
LinkedList additionally supports addFirst(),
 addLast(),removeFirst(),  removeLast() etc. 

ArrayList is better for search and accessing elements
LinkedList is better for frequent insertions 
 and deletion operations. 
 
 LinkedList implements List interface, Queue and Deque. 
 
  Queue - FIFO   - insertion at the end of the queue. 
  Deque - Double edged queue 
  
  Queue: 
  -------
  offer() - adding element
  poll() remove the first element
  peek()  view first element
  
  Deque: 
    - pollFirst() - remove first element
    -pollLast() remove last element
    getFirst() retrieves first element
    addFirst()   insert element at begining
    addLast()    insert element at End
  
 
 Set operations: 
 ----------------
 - Set is an interface in java.util package which extends
  Collection interface.  
- Set is a type of collection which does not allow duplilcate elements.
- It does not maintain insertion order for some implementations like
   HashSet
- Set interface is implemented by classes like HashSet, LinkedHashSet,
 and TreeSet classes. 
- Only one null is allowed 


HashSet is an implementation of Set interface
HashSet does not maintain insertion order
It does not allow duplicates
It allows a null object
It allows operations like Union and Intersection
addAll(Set s) method is used to combine the elements of 2 sets.
 but duplicates will be removed
retainAll(Set s) method is used for intersection oepration, i.e.
common elements present in both sets. 

removeAll():
Removes all elements that present in another collection. 

LinkedHashSet: 
Both HashSet and LinkedSet implements Set interface
Both do not allow duplicates
Main difference is about the ordering of the elements
HashSet does not maintain insertion order, where as LinkedHashSet
maintains insertion order.
Insertion order is preserved using LinkedHashSet,

If you want to maintain  unique elements in a Set and 
also want the insertion order to be preserved then 
LinkedHashSet  is the best option. 

LinkedHashSet internally maintains Hashtable for  fast  searching
LinkedHashSet internally implements doubly linked list to
maintain insertion order.
