try with resources(java 7)
--------------------------
try with resources allows you to declare resources like files, streams, database connections
inside the try statement so that they are automatically closed after use. 

- It eliminates the use for a finally block to close resources
- prevents resource leaks
- try with resources works with any object that implements AutoCloseable interface 

Syntax: 

try(FileWriter writer = new FileWriter("out.txt")){

//statements

}catch(Exception e){e.printStackTrace()}

AutoCloseable interface: 
=========================
AutoClosable interface enables automatic resource management. 
- its in java.lang
- contains a single method close()

Any class that implements Autoclosable can be used in
try-with-resources
eg: 
FileWriter,FileReader,BufferedWriter,BufferedReader, Connection etc. 

- When you use try-with-resources, JVM will automatically call close()
- Even if exception occurs, resources are still safely closed.

BufferedWriter and BufferedReader classes: 
- These classes adds a buffer(temporary memory) to speed up the 
    reading and writing.
- Buffer is a temporary memory area used to store data while reading or writing, 
- All the characters are first read/written to a buffer and then all characters
  are read/written at a time instead of one character a time. 
  
- In BufferedReader/BufferedWriter, the buffer improves the performance by
  reducing direct I/O calls . 
- read/wite happens in chunks instead of repeatedly accessing the file.  


BufferedReader class: 
- BufferedReader uses internal buffer
- It reads a chunk of data from the file into memory once. 
- when call readLine() or read() method the data is reterieved 
  from the buffer in memory.
- data is retrieved until the buffer is empty. 
- readLine() method returns null once it reaches the end of file.

Serialization: 
- Serialization is the process of converting an object into a byte stream
- Serialization is used to persist(saves the state) the object even 
  after the program exigs. 
- The converted byte stream can be saved as a file or sent over a network.
- The byte stream can be sent to a different machine. 
- Generally the bytestream is stored in a file with .ser extension. This
  file is platform independent. 
  
  Steps for Serialization: 
  - create a n object of FileOutputStream
  - Pass the name of the file(String) as an argument to FileInputStream constructor
  - Wrap it with BufferedOutputStream - for faster writing
  - Wrap with ObjectOutputStream which converts the object into bytes. 
  - Use writeObject(object) to save the object. 
  
  
  Steps for Deserialization: 
  -  Deserialization is the process of converting byte stream into an object. 
  -  Create an object FileInputStream
  - Pass the name of the file(String) as an argument to FileOutputStream constructor. 
  - Wrap it with BufferedInputSTream for faster reading
  - Wrap with ObjedtInputStream which rebuilds the object from bytes. 
  - Use readObject() to get the originbal object.

  NotserializableException: 
  ------------------------
  NotSerializableException is a runtime exception. 
  It occurs when we try to serialization an obect that does not implement Serializable
   interface. 
  For an object of a class to be eligible for serialization process, the class
   must implement Serializable marker interface. 
   
 
   
  transient modifier:
  ==================
  
  transient is a non-access modifier 
  It is used to exclude a variable from serialization process. 
  
  When an object is seriallized into a bytestream all of its variables are automatically
  saved. 
  
  But when we want a particular field to skip the serialization process, we 
  need to use the modifier 'transient' for that field. 
  
  Eg; Passwords, secret keys or some confidential data.
  
  transient modifier only works with instance variables.
  
 transient modifier with static variables: 
 -----------------------------------------
 static variables are not serialized at all
 Serialization works on objects
 static variables belong to the class, but not individual objects
 so java does not save static variables when serializing an object.
